## Назначение

Основной продукт — точка сборки всех смыслов. Берём всё наработанное (исследование, ядро оффера, распаковку, сервис, безопасность) и собираем в единый документ. Из него потом пишется лендинг.

---

## Шаг 1: Генерация бонусов

### [TECHNIQUE] Побочные явления = бесплатные бонусы

Всё, что создаётся в процессе разработки продукта, уже является бонусом:
- Дашборд/конструктор → "готовый дашборд с полной информацией"
- Список вопросов для исследований → "список готовых вопросов для CustDev"
- Шаблоны инструментов → "готовый шаблон в [инструменте]"
- Структура текста/скрипта → "пошаговая структура продающего текста"

### [TECHNIQUE] Специально созданные бонусы
- Дополнительные спикеры
- Гайды, чек-листы, конструкторы, схемы
- Маленькие инфопродукты по тематике

### [PATTERN] Постоянный список бонусов
Вести отдельный постоянный список. Увидел идею — записал. Бонусы используются многократно: в tripwire, в лид-магнитах, в модулях, в тарифах.

### [PATTERN] Привязка к модулям
Каждый бонус привязывается к конкретному модулю по тематике. Бонус к модулю "настройка рекламы" не может быть "чек-лист поиска клиентов" — нарушает логику.

## Шаг 2: Сервис и кайфовость

### [TECHNIQUE] Тест "И что?"

Каждый тезис о сервисе проверяется вопросом "И что? Что я от этого получу?" Если ответа нет — тезис сухой и не работает.

```
❌ "На курсе будут групповые и личные разборы"

✅ "На курсе будут групповые и личные разборы.
   На групповых — случайные ученики, ты обучишься на их ошибках.
   На личных — трекер разберёт конкретно твою ситуацию."
```

### Элементы сервиса (язык выгод)

| Элемент | Как описать |
|---------|-------------|
| Короткие уроки | Не часовые зумы, а короткие и лёгкие |
| Текстовые конспекты | Не надо копаться в видосах — поиск по тексту |
| Домашние задания | Каждое плавно подводит к результату |
| Проверка ДЗ | Индивидуальная обратная связь, устранение ошибок |
| Комьюнити | Закрытый чат с N опытными [специалистами] |
| Психолог (премиум) | Сессии для работы с сомнениями и мотивацией |

## Шаг 3: Безопасность

### [CHECKLIST] Блоки безопасности

1. **Регалии спикера** — только то, что влияет на решение. Без "хожу сквозь огонь и играю на пианино"
2. **Кейсы учеников** — массив, не 3-5 отзывов. Идеал: ссылка на канал "листай, пока рука не отсохнет"
3. **Статистические исследования** — данные от крупных платформ (рост рынка, спрос на профессию)
4. **Информация о кураторах** — описание, опыт, специализация
5. **Киллер-фичи и железобетонные гарантии**

### [TECHNIQUE] Сбор кейсов (метод Сердитова)
- Просить учеников скидывать результаты в чат (мотивация: признание/бонус)
- Сваливать все кейсы в отдельный телеграм-канал
- На лендинге: "Перейди по ссылке и посмотри результаты. Листай, пока рука не отсохнет."

## Шаг 4: Модули

### [PATTERN] Два типа структуры

| Тип | Когда | Пример |
|-----|-------|--------|
| **Элементы системы** | Нет хронологического порядка. Каждый элемент даёт мини-результат | Курс по таргету: воронки, связки, продажи — можно в любом порядке |
| **Пошаговая схема** | Одно не может жить без другого | Нутрициология: анализы → расшифровка → рекомендации |

### [ANTI-PATTERN] Скучные названия модулей

```
❌ "Модуль 1. Воронки"
❌ "Модуль 2. Reels"
✅ "Настройка рекламы за 15 минут"
✅ "Высококонверсионные креативы"
```

Каждое название должно звучать как отдельный самостоятельный продукт.

### [FORMULA] Описание модуля

```
[Боль/проблема]
    ↓
[Почему существующие решения не работают]
    ↓
[Что конкретно получишь после модуля] — на языке выгод
```

### [PATTERN] Бонусы к модулям
Минимум 1 бонус на модуль. Бонус соответствует тематике. Бонусы не нужно детально расписывать — пара уточняющих фактов.

## Шаг 5: Тарифы

### [FORMULA] Структура тарифной сетки

```
Тариф 1 (базовый)    Тариф 2 (средний)    Тариф 3 (премиум)
─────────────────    ──────────────────    ──────────────────
Модули 1-3           Модули 1-5            Все модули
Без обратной связи   Кураторы              Личная обратка от спикера
Базовые бонусы       + доп. бонусы         + все бонусы
```

Отличия строятся по 3 осям:
1. **Доступ к модулям**
2. **Уровень обратной связи**
3. **Бонусы**

## Шаг 6: Ценообразование

### [FORMULA] Цена = Рынок + Внутренний пульс + Математика

**1. Пульс рынка:**
- Смотреть на конкурентов **своего уровня** (не медийных — у них надбавка за бренд)
- Сравнивать точку А, не точку Б

**2. Контекст аудитории:**
- Новички → не пугать ценой, брать количеством
- Опытные → можно дорого
- Премиум → глупо ставить мало, когда хвастаются расходами на обучение

**3. Внутренний пульс:**
- Цена должна быть комфортной для тебя, иначе саботаж продаж
- Опыт снимает страх постепенно

**4. Математика:**
```
Цена × Кол-во учеников > Все расходы + Желаемая прибыль
```

### [PATTERN] Скидки: 40%+ или ничего

| Работает | Не работает |
|----------|-------------|
| Скидка 40-50%+ | Скидка 5-10% |
| Зачёркнутая старая цена + новая | Маленькие скидки в 5000₽ |

Скидки меньше 40% не цепляют и не влияют на решение.

## Чек-лист сборки

### [CHECKLIST] Основной продукт

- [ ] Нагенерены бонусы (побочные + специальные)
- [ ] Ведётся постоянный список бонусов
- [ ] Сервис описан на языке выгод (тест "и что?" пройден)
- [ ] Безопасность: регалии, кейсы (массив), исследования, кураторы, гарантии
- [ ] Определён тип структуры модулей (элементы / пошаговая)
- [ ] Каждый модуль назван как отдельный продукт
- [ ] Каждый модуль: боль → решение → результат
- [ ] К каждому модулю привязаны бонусы по тематике
- [ ] Тарифная сетка (модули + обратная связь + бонусы)
- [ ] Ценообразование: рынок + внутренний комфорт + математика
- [ ] Скидки ≥40% или не делать вообще
