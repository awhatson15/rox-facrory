Пишу статьи в стиле Макса Галсона для galson.pro. От исследования до публикации без согласования.

**Принцип скользкой горки:** Каждое предложение тянет к следующему. Не информационная каша, а история с позицией.

## Быстрый старт

Создание статьи от идеи до публикации:

```bash
# Полный пайплайн статьи  
"напиши статью про [ТЕМА]"

# Только исследование темы
"исследуй тему: [ТЕМА] для статьи"

# Только написание по готовому ресерчу
"напиши статью по этому материалу: [ДАННЫЕ]"
```

**ОБЯЗАТЕЛЬНО загружаются файлы стиля перед написанием.**

## Частые ошибки

❌ **"AI" вместо "ИИ"** — только в названиях продуктов  
❌ **Стрелки → в тексте** — использовать тире –  
❌ **"Сегодня хочу поговорить"** — сразу в суть  
❌ **Пафос про революцию ИИ** — избегать клише  
❌ **Длинные вступления** — хук в первом предложении  
❌ **Выдуманные цифры** — только проверенные факты  
❌ **Нет позиции автора** — Макс всегда имеет мнение  
❌ **Пассивный залог** — активный и личностный тон

### Голос Макса (КРИТИЧНО)

**✅ Правильно:**
- "Я попробовал", "Я застрял" — первое лицо
- Конкретные цифры: "1.5 миллиона за 5 дней"
- Разговорный тон: "Короче", "Кайф?", "Честно?"
- Валюта справа: 100$, 1490₽
- **ИИ**, не AI (кроме названий)
- Абзацы 1-3 предложения
- Позиция по теме

**❌ Неправильно:**
- Нейтральный пересказ
- Восклицательные знаки
- "Это не X. Это Y." — нейросетевой паттерн
- Пафосные заявления
- Stripe (в России не используется)
- "Кстати..." перед CTA

### Структура статьи

**Новостная/трендовая:**
1. **ХУК** — факт/цифра/ситуация (1-3 предложения)
2. **КОНТЕКСТ** — что случилось, когда, кто (2-3 абзаца)
3. **РАЗБОР** — основная часть с H2/H3, скриншотами
4. **ПОЧЕМУ ВАЖНО** — связь с читателем (1-2 абзаца)
5. **ЧТО ДЕЛАТЬ** — практические шаги (список)
6. **ФИНАЛ** — итоговая мысль с эмоцией

**Экспертная/обзорная:**
1. **ХУК** — провокационный тезис
2. **ПРОБЛЕМА** — что не так с текущим подходом
3. **РЕШЕНИЕ** — как работает технология
4. **ОПЫТ** — "я попробовал и вот что вышло"
5. **ПРАКТИКА** — конкретные шаги
6. **ФИНАЛ** — позиция + вопрос

### SEO оптимизация

**Frontmatter:**
```yaml
---
title: "Заголовок ≤60 символов с ключевым словом"
description: "Описание ≤160 символов"
date: YYYY-MM-DD
readingTime: "X мин"
tags: ["ИИ тег", "тема", "OpenClaw"]
keywords: "ключевые, слова, через, запятую"
cover: "/blog/<slug>/cover.jpg"
---
```

**В тексте:**
- H2 заголовки с ключевыми словами
- Ключевое слово в первом абзаце
- Внутренние ссылки (1-3)
- Внешние ссылки на источники (2-5)
- Alt-тексты у изображений

### Скриншоты через API

**microlink.io (основной способ):**
```bash
curl -s "https://api.microlink.io/?url=https://example.com&screenshot=true&waitForTimeout=5000" \
  | python3 -c "import sys,json,base64,urllib.request; d=json.load(sys.stdin); urllib.request.urlretrieve(d['data']['screenshot']['url'], '/tmp/screenshot.png')"

# Конвертация PNG→JPEG
sips -s format jpeg /tmp/screenshot.png --out output.jpg
```
